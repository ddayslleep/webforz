# CSS核心布局知识点

## 一. 盒子模型
所有HTML元素都可看作盒子？

**所有元素在CSS渲染时都生成一个或多个矩形盒子**，但元素与盒子不是一一对应。例如：

- 内联元素（`<span>`）可能被拆成多行，每行是一个“行框盒子”
- `display: none` 的元素不生成任何盒子

### 1. 盒子模型四大组成部分
盒子从内到外依次为：**内容区域**、**内边距**、**边框线**、**外边距**，核心CSS属性对应如下：
```css
div{
    width: 200px;  
    height: 200px;
    border: 1px solid #000;
    background-color: pink; 
    padding: 20px; 
    margin: 50px;  
}
```
### 2. 边框线（border）
#### （1）边框样式分类
- `solid`：实线（最常用）
- `dashed`：虚线
- `dotted`：点线
#### （2）统一设置边框
语法：`border: 边框宽度 边框样式 边框颜色;`
示例：
```css
border: 1px solid #000;
border: 1px dashed #4e1c1c;
border: 1px dotted #000;
   /*单独设置单方向边框 */
border-top: 2px solid red;    /* 上边框 */
```
### 3. 内边距（padding）
#### （1）核心特点
与边框线类似，支持单方向、多值批量设置，多值遵循**顺时针规则**（上→右→下→左），**==缺省值取对角值==**。
#### （2）多值写法规则
```css
padding: 10px 20px 30px 40px; 
padding: 10px 20px 30px;     
padding: 10px 20px;         
padding: 10px;              
```

### 4. 尺寸计算
盒子实际占据的宽/高 = 内容宽/高 + 内边距*2 + 边框宽度*2，解决盒子被撑大的两种方式：
1. **手动减法**：根据实际需求，手动计算并调整内容区域的宽/高；

2. **内减模式**：给盒子设置`box-sizing: border-box;`，此时CSS设置的`width/height`直接表示盒子实际宽/高，内容区域会自动收缩，无需手动计算（**推荐使用**）。

   ==box-sizing: border-box;可一定程度上避免布局混乱 eg：案例1 ->演示==
### 5. 版心居中
让盒子在父容器中水平居中，==**核心条件**：盒子必须设置固定宽度==，然后给盒子添加`margin: 0 auto;`。

**`margin: 0 auto;` 为啥能水平居中？能否垂直居中呢？**

**原理**：块级元素在水平方向上，`auto` 会均分剩余空间。设置左右 `auto` + 固定宽度 → 左右平分剩余空间 → 居中。

**垂直居中不可以**，因为块级元素在垂直方向默认由内容撑开，没有“剩余空间”的概念。

示例：

```css
div{
    width: 200px;
    height: 200px;
    background-color: pink;
    margin: 0 auto; /* 水平居中，垂直方向外边距为0 */
}
```
### 6. 清除默认样式
浏览器会给部分HTML元素设置默认的margin、padding（如body、ul、p），或默认样式（如li的项目符号），开发前需统一清除，避免布局错乱，**通用清除样式**：

有浏览器默认样式

```css
/* 通配符选择器，清除所有元素的默认内外边距，开启内减模式 */
*{
    margin: 0;
    padding: 0;
    box-sizing: border-box; /* 避免盒子被内边距、边框撑大 */
}
/* 清除列表的默认项目符号 */
li{
    list-style: none;
}
```


### 7. 元素溢出处理

当盒子内的内容超出盒子的宽/高时，通过`overflow`属性控制内容的显示方式，核心取值及特点：
- `scroll`：始终显示滚动条（包含水平、垂直滚动条），即使内容未溢出；
- `auto`：**推荐使用**，内容溢出时显示对应滚动条，未溢出时不显示滚动条（无水平滚动条）；
- `hidden`：隐藏溢出的内容；
- `visible`：默认值，溢出内容直接显示在盒子外部。
### 8. 外边距的特殊问题

#### example2 演示

#### （1）外边距合并
两个相邻的块级元素，垂直方向的外边距会发生合并，**最终取两个外边距中的较大值**（水平方向外边距无合并问题）。
#### （2）外边距塌陷（父子级）
父子级标签中，给子元素添加**上外边距**，会导致父元素一起被向下推动，即“子元素拖父元素下水”，**解决方案（推荐padding-top法）**：

**为什么会出现塌陷问题？**

**浏览器这么设计有什么用**？

举个最直观的例子：

```
<p>第一段</p>
<p>第二段</p>
```

- p 标签默认 `margin: 1em 0`
- 上下各 1em
- 如果不塌陷，两段之间会变成 **2em**，间距太大，很难看

浏览器为了**正常排版文章**，就规定：

**垂直相邻块级 margin 只取最大值**。



### 9. 行内元素的内外边距问题

行内元素分两类，盒模型表现完全不同：

1. 非替换行内元素

   （如<a> <span>）

   - 水平方向：`margin`、`padding` 完全生效，正常占据布局空间
   - 垂直方向：`margin-top/bottom` **完全无效**；`padding-top/bottom` 仅视觉撑开背景 / 边框，**不影响行高、不改变布局**，可能与上下行内容重叠

   

2. 替换行内元素

   （如 <input> <img><button>)

   - 水平 + 垂直方向：`margin`、`padding` 完全正常生效，与块级元素规则一致

   

**常见问题与解决方案**

1. 非替换行内元素垂直边距失效

   若需完整支持垂直 margin/padding，将其 display 改为 inline-block

2. 行内元素垂直居中

   - 传统方案：给父容器设置 `line-height` 等于容器高度，实现行内元素（如导航栏 `<a>`）垂直居中
   - 现代方案：给父容器加 `display: flex; align-items: center`，兼容性更好、更灵活

   

   **总结**

非替换行内元素垂直 `margin` 无效、`padding` 仅视觉生效；替换行内元素边距完全正常。需垂直边距用 `inline-block`，垂直居中用 `line-height` 或 `flex`。

### 10. 圆角效果（border-radius）

CSS 圆角**不是给元素加了圆角边框**，而是==**用一个「圆角矩形」作为裁剪路径（clip path），把元素超出这个路径的部分直接裁掉**。==

这个圆角矩形的四个角，是**以对应角为圆心、`border-radius` 为半径的四分之一圆弧**，圆弧内侧保留，外侧全部裁剪。

#### （1）核心规则
多值写法与padding一致，**从左上角开始顺时针**（左上→右上→右下→左下），**缺省值取对角值**，单位支持`px`/`%`。
#### （2）基础写法
```css
div{
    margin: 50px auto;
    width: 100px;
    height: 100px;
    background-color: blue;
    border-radius: 10px 20px 30px 40px; /* 四值：左上 右上 右下 左下 */
    /* 单值：四个角圆角相同 */
    /* border-radius: 20px; */
}
```
#### （3）常用圆角形状
1. **正圆效果**：盒子为**正方形** + `border-radius: 50%;`（若盒子非正方形，会变成椭圆）；
   ```css
   img{
       width: 200px;
       height: 200px; /* 正方形 */
       border-radius: 50%; /* 正圆 */
   }
   ```
2. **胶囊形状**：盒子为长方形 + `border-radius`取值为**盒子高度的一半**（或略小于高度的一半）；
   ```css
   div{
       width: 200px;
       height: 100px; /* 长方形 */
       background-color: powderblue;
       border-radius: 40px; /* 胶囊形，取值接近高度的一半 */
   }
   ```
### 11. 阴影效果（box-shadow）
给盒子添加阴影，不占用实际布局空间，语法：`box-shadow: 水平偏移 垂直偏移 模糊距离 阴影颜色;`
- 水平偏移：正值向右，负值向左；
- 垂直偏移：正值向下，负值向上；
- 模糊距离：值越大，阴影越模糊（不能为负）；
- 阴影颜色：推荐使用`rgba()`设置半透明，更贴合实际开发需求。
示例：
```css
div{
    width: 200px;
    height: 100px;
    background-color: powderblue;
    border-radius: 40px;
    box-shadow: 20px 20px 10px rgba(0,0,0,0.5); /* 右下方向的半透明阴影 */
}
```

## 二. 浮动（float）

**浮动的特性**

- 元素脱离正常文档流，不占用原位置（“脱标”）。
- 块级元素可并排排列，不再独占一行。
- 内联元素或文本会环绕浮动元素。
- 父容器高度会塌陷（因为浮动子元素不参与高度计算）。

**常见场景**

- 文字环绕图片（如新闻配图左/右浮动）。
- 横向导航菜单（li 浮动实现一行排列）。
- 两列/三列布局（浮动配合清除浮动）。
- 卡片列表（浮动实现瀑布流效果）。

**注意**：浮动是布局工具，脱标不是错误；为避免布局错乱，需在父容器使用 `clearfix` 或 `overflow: hidden` 清除浮动。

### 核心作用
让**块级元素在同一行显示**，实现横向布局，是早期网页布局的核心方式（如产品列表、导航栏）。
### 1. 基础属性
- `float: left;`：元素向左浮动，**顶对齐**（同一行的浮动元素顶部对齐）；
- `float: right;`：元素向右浮动。



### 2 .清除浮动
#### （1）清除浮动的原因
父盒子若未设置固定高度，子元素全部浮动后，父盒子会**高度塌陷**（高度为0），因此需要清除浮动，让父盒子能包裹住浮动的子元素。
#### （2）四种清除浮动的方法
##### 方法1：额外标签法（最基础，不推荐）
- 步骤：在父级元素内容的**最后面**添加一个**空的块级元素**（如`<div class="clearfix"></div>`）；
- 给该空元素设置`clear: both;`（清除左右两侧的浮动影响）。
代码示例：
```css
.top {
    margin: 10px auto;
    width: 1200px;
    background-color: pink; /* 父盒子未设高度 */
}
.left { float: left; width: 200px; height: 300px; background-color: orange; }
.right { float: right; width: 950px; height: 300px; background-color: skyblue; }
.bottom { height: 100px; background-color: darksalmon; }
/* 清除浮动的样式 */
.clearfix { clear: both; }
```
```html
<div class="top">
    <div class="left"></div>
    <div class="right"></div>
    <!-- 额外添加的块级元素，放在最后 -->
    <div class="clearfix"></div>
</div>
<div class="bottom"></div>
```
##### 方法2：单伪元素法（推荐，简洁）
利用CSS伪元素`::after`替代额外的HTML标签，实现无侵入式清除浮动，**给父盒子添加类名clearfix**。
```css
/* 单伪元素清除浮动 */
.clearfix::after {
    content: ""; /* 必须有，内容为空 */
    display: block; /* 必须设为块级元素 */
    clear: both; /* 清除左右浮动 */
}
```
```html
<!-- 父盒子添加clearfix类 -->
<div class="top clearfix">
    <div class="left"></div>
    <div class="right"></div>
</div>
```
##### 方法3：双伪元素法（最优推荐）
在单伪元素法的基础上，增加`::before`伪元素，**同时解决浮动高度塌陷和父子级外边距塌陷问题**，给父盒子添加类名clearfix即可。
```css
/* 双伪元素清除浮动 + 解决外边距塌陷 */
.clearfix::before,
.clearfix::after {
    content: "";
    display: table; /* 合并相邻的外边距 */
}
.clearfix::after {
    clear: both; /* 清除浮动 */
}
```
##### 方法4：overflow法（快速临时解决）
给**父盒子**添加`overflow: hidden;`，快速实现清除浮动，缺点：会隐藏盒子内溢出的内容，适合简单布局临时使用。
```css
.top {
    margin: 10px auto;
    width: 1200px;
    background-color: pink;
    overflow: hidden; /* 清除浮动 */
}
```

## 三. Flex布局（弹性布局）

#### ==Example4演示==

### 核心作用

现代网页布局的**首选方式**，实现弹性、灵活的横向/纵向布局，**不会出现脱标问题**，适配各种屏幕尺寸，替代传统浮动布局的繁琐操作。
### 1. 核心组成
给父盒子设置`display: flex;`，该父盒子成为**弹性容器**，其子元素自动成为**弹性项**；弹性容器会让弹性项自动挤压或拉伸，适应容器空间。
### 2. 主轴与侧轴
Flex布局的核心是**主轴**和**侧轴**，二者相互垂直：
- **主轴**：弹性项的排列方向，默认**水平从左到右**；
- **侧轴**：与主轴垂直的方向，默认**垂直从上到下**；
- 可通过`flex-direction`修改主轴方向，侧轴会自动跟随变化。
### 3. 主轴对齐方式（justify-content）
控制弹性项在**主轴方向**的对齐与间距，核心取值（默认主轴水平）：
- `flex-start`：默认值，弹性项靠主轴起始端对齐；
- `flex-end`：弹性项靠主轴结束端对齐；
- `center`：弹性项在主轴方向**居中对齐**（常用）；
- `space-between`：**两端对齐**，弹性项之间的间距均分，无左右两端间距（常用）；
- `space-around`：弹性项之间的间距为**两端间距的2倍**，间距均匀分布。
### 4. 侧轴对齐方式（align-items）
控制弹性项在**侧轴方向**的对齐，核心取值（默认侧轴垂直），给弹性容器设置：
- `flex-start`：弹性项靠侧轴起始端对齐；
- `flex-end`：弹性项靠侧轴结束端对齐；
- `center`：弹性项在侧轴方向**居中对齐**（常用，实现垂直居中）；
- `stretch`：默认值，弹性项在侧轴方向**自动拉伸**（**前提：弹性项未设置侧轴方向的尺寸**，如垂直侧轴未设置height）。
#### 单独设置单个弹性项的侧轴对齐
给指定弹性项设置`align-self: 对齐方式;`，覆盖容器的`align-items`，实现差异化对齐：
```css
.box {
    display: flex;
    align-items: flex-end; /* 容器侧轴靠下对齐 */
    height: 300px;
    border: 1px solid black;
}
/* 倒数第二个弹性项侧轴居中对齐 */
.box div:nth-last-child(2) {
    align-self: center;
}
```
### 5. 修改主轴方向（flex-direction）
核心取值，实现横向/纵向布局的切换：
- `row`：默认值，主轴**水平从左到右**；
- `row-reverse`：主轴水平从右到左；
- `column`：主轴**垂直从上到下**（常用，实现纵向布局）；
- `column-reverse`：主轴垂直从下到上。
#### 经典应用：盒子内元素水平+垂直居中
修改主轴为垂直方向，配合主轴和侧轴居中，实现子元素在父盒子中完全居中：
```css
.box {
    display: flex;
    flex-direction: column; /* 主轴改为垂直方向 */
    justify-content: center; /* 主轴（垂直）居中 */
    align-items: center;     /* 侧轴（水平）居中 */
    width: 150px;
    height: 120px;
    background-color: pink;
}
img {
    width: 32px;
    height: 32px;
}
```
### 6. 弹性伸缩比（flex）
控制弹性项在**主轴方向**的占比，实现弹性项的自动拉伸/收缩，**无单位**，表示弹性项之间的比例关系。
#### 核心规则
1. 默认情况下，弹性项的主轴尺寸靠内容撑开；
2. 给弹性项设置`flex: 数值;`，弹性容器的剩余空间会按比例分配给弹性项；
3. 若所有弹性项都设置flex，容器空间会按比例全部分配。
示例：
```css
.box {
    display: flex;
    flex-direction: column; /* 主轴垂直，按高度分配 */
    height: 300px;
    border: 1px solid black;
}
/* 三个弹性项高度比例为1:2:1 */
.box div:nth-last-child(1) { flex: 1; }
.box div:nth-last-child(2) { flex: 2; }
.box div:nth-last-child(3) { flex: 1; }
```
### 7. 弹性盒子换行（flex-wrap）
默认情况下，弹性容器会强制所有弹性项在**同一行显示**（即使宽度不足），通过`flex-wrap`实现换行：
- `flex-wrap: nowrap;`：默认值，不换行，弹性项自动挤压；
- `flex-wrap: wrap;`：**自动换行**（常用），宽度不足时，弹性项换行显示，形成多行布局。
### 8. 多行对齐（align-content）
控制**多行弹性项**在主轴方向的整体对齐与间距，**对单行弹性项无效**，核心取值与`justify-content`一致（`center`/`space-between`/`space-around`等），给弹性容器设置即可。

## 四. Grid 布局（网格布局）

#### **==Example5演示==**

### 核心作用

**二维布局神器**，可以**同时控制行和列**，适合做：网页整体布局、卡片墙、宫格、表单、复杂排版。

比 Flex 更强大，Flex 是一维，Grid 是二维。

### 1. 核心组成

给父盒子设置`display: grid;`，父盒子成为**网格容器**，子元素自动成为**网格项**。

### 2. 行与列（二维轴线）

Grid 直接分**行（row）** 和**列（column）**，不用像 Flex 那样区分主轴侧轴。

- **水平方向 = 列**
- **垂直方向 = 行**

### 3. 定义行列数量与宽度（最核心）

```
{
  grid-template-columns: 列宽1 列宽2 列宽3;  /* 定义几列、每列多宽 */
  grid-template-rows: 行高1 行高2;          /* 定义几行、每行多高 */
}
```

#### 超级常用写法

- `1fr`：按**比例**分配剩余空间（Grid 专用，最强单位）
- `repeat(数量, 尺寸)`：重复生成
- `minmax(最小, 最大)`：限制范围
- `auto-fit` 响应式自适应（不用媒体查询）

```
/* 3列等宽 */
grid-template-columns: repeat(3, 1fr);

/* 响应式卡片自适应 */
grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
```

### 4. 设置格子间距（gap）

```
gap: 10px;           /* 行列间距一样 */
row-gap: 10px;       /* 行间距 */
column-gap: 15px;    /* 列间距 */
```

### 5. 网格对齐方式（和 Flex 非常像）

Grid 对齐分两套，很好记：

#### ① 整个网格在容器里对齐（内容少的时候）

- `justify-content`：**水平**对齐（左右）
- `align-content`：**垂直**对齐（上下）

#### ② 网格项在格子内对齐（常用）

- `justify-items`：项目**水平**对齐
- `align-items`：项目**垂直**对齐
- `place-items: center;` → **水平 + 垂直全部居中（一句搞定）**

取值：`start / end / center / stretch`

### 6. 单独控制某个网格项对齐

```
 {
  justify-self: 水平对齐;
  align-self: 垂直对齐;
  place-self: center;
}
```

### 7. 合并单元格（跨行列，最强功能）

让一个盒子**占多列 / 多行**，Flex 做不到，Grid 轻松实现。

```
{
  grid-column: 起始列 / 结束列;   /* 占几列 */
  grid-row: 起始行 / 结束行;      /* 占几行 */
}
```

```
grid-column: 1 / 3;   /* 从第1列跨到第3列 = 占2列 */
grid-column: span 2;  /* 直接跨2列 */
```

### 8. 区域命名（像拼图一样布局）

直接给区域起名，实现头部、侧边栏、主体、底部布局：

```
{
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
}
.header { grid-area: header; }
```

### 9. 自动换行 / 隐式网格

项目超出定义行列时，自动生成新行 / 列：

```
grid-auto-rows: 100px;   /* 自动生成的行高 */
grid-auto-flow: row;     /* 自动排列方向 */
```



## 五. 定位（position）

### 核心作用
实现元素的**精准定位**（如轮播图箭头、悬浮导航、弹窗），让元素脱离标准流/浮动流，自由摆放在页面的任意位置。
### 定位的通用属性
定位元素通过**偏移属性**控制位置：`top`/`bottom`（垂直偏移）、`left`/`right`（水平偏移），需配合定位模式使用才生效。
### 1. 相对定位（position: relative;）
#### 核心特点
1. **参照物**：元素**自己原来的标准流位置**；
2. **脱标与否**：**不脱标**，仍占用标准流的位置，不会影响其他元素；
3. **显示模式**：标签的显示模式特点不变（如行内元素宽高仍无效）；
4. **应用场景**：作为绝对定位的父容器（子绝父相），或微调元素的位置。
### 2. 绝对定位（position: absolute;）
#### 核心特点
1. **参照物**：先找最近的**已定位的祖先元素**（设置了relative/absolute/fixed的祖先），若无则参照**浏览器窗口**；
2. **脱标与否**：**脱标**，不占用标准流的位置，会覆盖其他元素；
3. **显示模式**：显示模式自动转变，**行内元素可设置宽高**（行内块特点）；
4. **核心应用**：`子绝父相`（子元素绝对定位，父元素相对定位），实现子元素在父元素内的精准定位（如轮播图箭头、商品标签）。

#### 经典示例：子绝父相实现商品标签

**绝对定位参照物**

- **规则**：`position: absolute` 的元素，参照**最近的非 `static` 定位祖先**（即 `relative`、`absolute`、`fixed` 或 `sticky`）。
- **无此类祖先时**：参照**初始包含块**（通常为浏览器视口）。
- **常见误区**：说“参照父级”不准确，必须父级或任意祖先**不是 `static`** 才生效。

**示例对比**：
- 父级 `position: relative` → 绝对定位子元素按父级偏移。
- 父级 `position: static`（默认）且无其他非 `static` 祖先 → 子元素相对视口偏移。

### 3. 定位居中（绝对定位元素的水平+垂直居中）
绝对定位元素无法通过`margin: 0 auto;`居中，**最优方案**：`left: 50%; top: 50%; + transform: translate(-50%, -50%);`，实现精准居中，无需知道元素的宽高。
示例：

```css
img {
    position: absolute;
    left: 50%; /* 元素左侧在浏览器/父容器50%位置 */
    top: 50%;  /* 元素顶部在浏览器/父容器50%位置 */
    transform: translate(-50%, -50%); /* 元素自身向左、向上移动50% */
}
```
### 4. 固定定位（position: fixed;）
#### 核心特点
1. **参照物**：始终是**浏览器的可视窗口**，滚动页面时，元素位置保持不变；
2. **脱标与否**：**脱标**，不占用标准流的位置；
3. **显示模式**：行内块特点，行内元素可设置宽高；
4. **应用场景**：页面固定导航、回到顶部按钮、京东/淘宝的固定侧边栏。
#### 用法
`position: fixed;` + 偏移属性（`top`/`bottom`/`left`/`right`），示例：
```css
.back-top {
    position: fixed;
    bottom: 50px;
    right: 50px;
    width: 50px;
    height: 50px;
    background-color: red;
}
```
### 5. 粘性定位（position: sticky;）

#### ==Example5演示==

#### 核心特点
1. 结合了**相对定位**和**固定定位**的特点，元素在滚动到指定位置前为相对定位，滚动后变为固定定位；
2. **不脱标**，占用标准流位置；
3. **应用场景**：页面滚动时的吸顶导航（如知乎、B站的导航栏）。
#### 用法
`position: sticky;` + 偏移属性（如`top: 0;`，表示滚动到顶部时固定）。

### 6.堆叠顺序（z-index）
#### 核心作用
控制**定位元素**（relative/absolute/fixed/sticky）的堆叠层级，解决定位元素的覆盖问题。
### 核心规则
1. 未设置z-index时，遵循**后来者居上**：后写的定位元素会覆盖先写的；
2. z-index的取值为**正整数/0/负整数**，**值越大，元素越靠上**；
3. 只有定位元素才能设置z-index，非定位元素无效；
4. 父子元素中，子元素的z-index再大，也无法覆盖父元素。
示例：
```css
div {
    position: absolute;
    width: 200px;
    height: 200px;
}
.box1 {
    background-color: pink;
    z-index: 1; /* 层级1 */
}
.box2 {
    background-color: skyblue;
    left: 100px;
    top: 100px;
    z-index: 2; /* 层级2，覆盖box1 */
}
```



## 六.对齐（vertical-align）
### 核心作用
控制**行内元素/行内块元素**（如img、span、input、字体图标）在垂直方向的对齐方式，解决图片与文字基线对齐导致的间隙问题。
### 核心取值
- `baseline`：默认值，基线对齐；
- `middle`：**垂直居中对齐**（常用）；
- `top`：顶部对齐；
- `bottom`：底部对齐。
### 经典应用
解决图片与文字同行时的底部间隙：
```css
img {
    vertical-align: middle; /* 图片与文字垂直居中 */
    /* 或设置vertical-align: top/bottom; */
}
```

## 七. 其他常用CSS属性
### 1. 透明度（opacity）
控制元素的整体透明度，取值范围**0~1**：
- `opacity: 0;`：元素完全透明（仍占用布局空间）；
- `opacity: 0.5;`：元素50%透明；
- `opacity: 1;`：元素完全不透明（默认值）。
**注意**：会让元素内的所有子元素一起透明，若仅需背景透明，使用`rgba()`。
### 2. 光标类型（cursor）
控制鼠标悬浮在元素上时的光标样式，核心取值：
- `default`：默认光标（箭头）；
- `pointer`：**手型光标**（常用，给按钮、a标签设置）；
- `text`：文本光标（竖线）；
- `wait`：等待光标（加载中）；
- `move`：移动光标（十字）。
示例：
```css
button {
    cursor: pointer; /* 鼠标悬浮为手型 */
}
```
