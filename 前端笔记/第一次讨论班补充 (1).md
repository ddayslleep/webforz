# 第一次讨论班补充

## 一.空白折叠

浏览器在渲染 HTML 页面时，会将代码中出现的**多个连续的空白字符**，**折叠（合并）为一个单个的空格**，这一特性就是**空白折叠**，是 HTML 的默认排版规则。

目的是：让开发者可以自由排版 HTML 代码（加空格、Tab、换行），而不会影响页面的最终显示效果，避免代码格式导致页面布局错乱。

#### 1.哪些空白字符会被折叠？

并非只有空格键，**所有空白类字符**连续出现时，都会被统一折叠，包括 3 种：

1. **普通空格**（键盘敲的`Space`键）；
2. **制表符**（键盘敲的`Tab`键，多个缩进符）；
3. **换行符**（键盘敲的`Enter`键，代码的换行）。

**示例**：以下 3 种写法，浏览器渲染结果**完全一致**，都是`你好 世界`（单个空格分隔）

```
<!-- 多个普通空格 -->
<div>你好   世界</div>
<!-- Tab制表符+换行符 -->
<div>你好	
世界</div>
<!-- 多次换行+空格 -->
<div>你好
    世界</div>
```

#### 2.空白折叠的**特殊场景**

1. 元素首尾的空白会被忽略

   元素内容开头或结尾的空白字符，会被浏览器直接折叠（删除），不会显示。

   

   示例：

   ```
   <div>  你好世界  </div>
   ```

    → 渲染为

   ```
   你好世界
   ```

   （首尾无空格）。

2. 标签之间的空白会被折叠

   两个 HTML 标签之间的所有空白，也会被折叠为一个空格（若标签为行内 / 行内块元素，会出现 “标签间间隙” 问题）。

   ```
   <span>1</span>
   <span>2</span>
   ```

   

   渲染为

   ```
   1 2
   ```

   （两个 span 之间有一个空格），这也是行内元素并排时出现间隙的核心原因。

#### 3.如何**取消空白折叠**（保留原始空白格式）

开发中若需要原样显示空格、Tab、换行（如显示代码、诗词、表格数据），可通过 3 种方法取消空白折叠，**按使用优先级排序**，兼顾灵活度和实用性：

#### 方法 1：CSS 设置`white-space`属性（**最推荐，灵活可控**）

给需要保留空白的元素添加`white-space`属性，通过不同取值控制空白处理方式，适配所有 HTML 元素（div/p/span/pre 等），核心取值如下：

| 属性值     | 空白处理                       | 自动换行                             | 适用场景                                                     |
| :--------- | :----------------------------- | :----------------------------------- | :----------------------------------------------------------- |
| `normal`   | **默认**，折叠所有空白         | 自动换行                             | 普通文本排版                                                 |
| `pre`      | 保留所有空白（不折叠）         | **不自动换行**（超出容器会横向滚动） | 显示简短代码 / 固定格式文本                                  |
| `pre-wrap` | 保留所有空白（不折叠）         | **自动换行**（超出容器会换行）       | 显示长代码 / 诗词 / 需要固定格式且自动换行的内容（**最常用**） |
| `pre-line` | 折叠空格 / Tab，**保留换行符** | 自动换行                             | 普通文本需要保留手动换行的场景                               |
| `nowrap`   | 折叠所有空白                   | **强制不换行**（超出容器单行显示）   | 导航栏 / 单行文本展示                                        |

**核心示例（推荐 pre-wrap）**：

```
.code {
    white-space: pre-wrap; /* 保留空白+自动换行 */
    border: 1px solid #000;
    width: 200px;
}
```

浏览器会**原样保留代码的缩进、换行**，且超出 200px 宽度时自动换行，不会横向滚动。

#### 方法 2：使用 HTML`<pre>`标签（**预格式化标签，原生不折叠**）

<pre>是 HTML 的预格式化标签，天生取消空白折叠，会原样渲染内容中的所有空格、Tab、换行，无需额外 CSS。

**缺点**：默认不自动换行，内容超出容器会横向滚动；且默认有等宽字体、上下外边距的样式。

```
<pre>
    床前明月光，
  疑是地上霜。
    举头望明月，
  低头思故乡。
</pre>
```

**优化**：给`<pre>`加`white-space: pre-wrap;`，解决不自动换行问题

```
pre {
    white-space: pre-wrap;
}
```

#### 方法 3：使用 HTML**空白实体字符**（**精准控制单个 / 多个空格**）

用浏览器能识别的**特殊字符代码**表示空格，这类字符不会被折叠，可精准控制空格的数量和类型，常用的有 2 种（行内文本微调间距首选）：

| 实体字符 | 名称       | 显示效果               | 适用场景                       |
| :------- | :--------- | :--------------------- | :----------------------------- |
| ` `      | 不换行空格 | 单个半角空格           | 行内文本需要多个连续空格的场景 |
| ` `      | 全角空格   | 一个**汉字宽度**的空格 | 中文排版的间距微调（更美观）   |

```
<!-- 3个连续半角空格 -->
<div>你好&nbsp;&nbsp;&nbsp;世界</div>
<!-- 1个全角中文空格 -->
<div>你好&emsp;世界</div>
```

### 4.补充：行内元素 “标签间间隙” 的解决思路

因空白折叠，行内 / 行内块元素（span/img/a）标签之间的空白会被折叠为一个空格，导致元素之间出现间隙，可通过以下方法解决：

1. **代码中删除标签间的空白**：将标签写在同一行，无换行 / 空格；
2. **父元素设置`font-size: 0;`**：折叠空白的本质是 “空白字符被渲染为空格”，将父元素字体大小设为 0，空格就会消失，子元素再单独设置`font-size`；
3. **使用 Flex 布局**：给父元素设置`display: flex;`，会自动消除行内元素的标签间间隙（**推荐，一举两得**）。

### 5.空白折叠核心总结

1. 定义：浏览器将**多个连续空白字符（空格 / Tab / 换行）** 折叠为**一个空格**，是 HTML 默认规则；
2. 特殊：元素首尾空白会被忽略，标签间空白会折叠为一个空格；
3. 取消折叠：首选**CSS `white-space: pre-wrap;`**，次选`<pre>`标签，行内微调用` / `；
4. 延伸：行内元素间隙是空白折叠的典型场景，Flex 布局可快速解决。

## 三. 伪元素 ::before / ::after 核心实战要点

#### 1. 默认显示模式：**行内元素（inline）**

- 和 `<span>`、`<a>`、`<img>` 一样
- **一行可以多个**
- **不能直接设置宽高**
- **上下 margin /padding 不生效**

这就是为什么很多时候写：

```
.box::before {
  content: '';
  width: 20px;
  height: 20px;
  background: red;
}
```

==**发现不显示！**==

原因：**行内元素设置宽高无效**。

------

#### 2. 想让伪元素显示、能设置大小？

```
display: block;
/* 或者 */
display: inline-block;
```

**实战经验：**

- 要单独占一行 → `block`
- 要跟文字同行 → `inline-block`
- 要做小图标、小箭头、小装饰 → 一律 `inline-block`

------

#### 3. content 到底是什么？为什么不能省？

##### （1）`content` 是伪元素的**灵魂**

- **没有 content，伪元素就不会渲染到页面上**
- 它是用来放 “内容” 的：文字、符号、空

##### （2）三种写法

```
/* 1. 写文字（最常用：加小标签） */
content: '新品';

/* 2. 空内容（只做样式，最常用！） */
content: '';

/* 3. 不写 → 伪元素直接消失 */
/* 什么都不写 */
```

#### 实战结论：

**哪怕你只需要一个色块、箭头、装饰，什么文字都不想加，也必须写：**

```
content: '';
```

**不写 → 不显示。**

------

#### 4. 最标准、最常用的伪元素写法

```
.box::before {
  content: '';            /* 必须写 */
  display: inline-block;  /* 必须写，才能设置宽高 */
  width: 6px;
  height: 6px;
  background: #f30;
  border-radius: 50%;
}
```

------

#### 5. 实战中伪元素一般用来干什么？

1. **小红点、小圆圈**
2. **小箭头（下拉框、导航、提示框）**
3. **小标签（热、新、荐、VIP）**
4. **装饰线、小竖线、小图标**
5. **清除浮动（clearfix）**

全部都要用到上面这两条规则：

- `content: ''`
- `display: inline-block / block`

------

#### 6. 一句总结

- **::before / ::after 默认是行内元素**
- **想设置宽高，必须转块级或行内块**
- **content 不能省，空内容也要写 `content: ''`**

## 四. name 属性里写中文

**在表单的 name 属性里写中文（如 name="用户名"），会导致：**

1. **URL 乱码**
2. **后端接收不到数据 / 接收乱码**
3. **不同浏览器表现不一致**
4. **不符合 HTTP 协议规范**

**value 用中文没问题，但 name 用中文 = 高风险！**

------

#### 1.为什么 `name="中文"` 会出问题？

因为：

#### **HTTP 协议规定：URL 只能是 ASCII 字符（英文 / 数字 / 符号）**

你提交表单时，浏览器会把 `name=值` 拼到 URL 里：

```
xxx.php?用户名=admin&passwd=123
```

**中文不能直接出现在 URL 中！**

浏览器必须把中文**编码**，但：

- 不同浏览器编码方式不一样
- 后端解码方式不一样
- 极易出现 **乱码、丢失、解析失败**

------

#### 2.会出现的具体问题（你一定会遇到）

##### 1>. **URL 变成一堆乱码（% E5%95......）**

```
xxx.php?%E7%94%A8%E6%88%B7%E5%90%8D=admin
```

可读性极差，调试困难。

##### 2.> **后端接收不到参数**

PHP/Java/Python 都可能：

- 识别不了中文 name
- 读到乱码 key
- 直接拿不到数据

##### 3.> **兼容性灾难**

Chrome / Edge / Firefox 对中文参数编码**不统一**

→ 同样的表单，不同浏览器结果不一样。

#### 3. **不符合 Web 标准**

W3C 规范明确建议：

**表单 name 属性必须使用英文字母、数字、下划线。**

------

**重点区分**：

✅ **value 可以是中文**

`value="用户名"` 只是显示默认值，**完全没问题**。

❌ **name 不推荐中文**

`name="用户名"` → **高危、不规范、必埋坑**

规则：

- **name = 英文**（给后端看、给协议看）
- **value = 可中文**（给界面看、给数据看）

## 五. 可继承属性

#### 1. 字体相关（font 系列）

- `font-family`：字体类型（如微软雅黑、Arial）
- `font-size`：字体大小（如 16px、1.2rem）
- `font-weight`：字体粗细（如 normal、bold、700）
- `font-style`：字体样式（如 normal、italic 斜体）
- `font-variant`：小型大写字母（如 normal、small-caps）
- `font`：字体复合属性（继承所有单个 font 属性的设置）

#### 2. 文本排版相关

- `color`：文字颜色（核心！子元素默认继承父元素文字色）
- `line-height`：行高（垂直居中、文本行间距的核心，可继承）
- `text-align`：文本水平对齐（left/center/right/justify，块级元素生效，子元素继承）
- `text-indent`：文本缩进（如 2em 首行缩进，可继承）
- `text-transform`：文本大小写（uppercase/lowercase/capitalize）
- `letter-spacing`：字符间距（字与字之间的距离）
- `word-spacing`：单词间距（英文有效，空格分隔的单词）
- `white-space`：空白处理（如 normal/pre，就是之前讲的空白折叠控制，可继承）

#### 3. 列表相关（仅针对列表子元素）

- `list-style-type`：列表项目符号类型（如 none、disc 圆点、decimal 数字）
- `list-style-position`：项目符号位置（inside/outside）
- `list-style-image`：自定义项目符号图片
- `list-style`：列表复合属性

#### 4. 其他高频可继承

- `cursor`：鼠标光标类型（如 pointer 手型、default 箭头，子元素继承光标样式）
- `visibility`：元素可见性（visible/hidden，**区别于 display:none**，继承后子元素可单独修改）
- `opacity`：透明度（0~1，子元素会继承父元素的透明度，且无法单独取消）
