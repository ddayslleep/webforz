### 1. Web API 基本认识

#### 1.1 作用和分类

* 作用：就是使用JS 去操作 html 和浏览器
* 分类：DOM（文档对象模型）、BOM（浏览器对象模型）

#### 1.2 什么是DOM

* DOM 是浏览器提供的一套专门用来操作网页内容的功能
* DOM 作用
  * 开发网页内容特效和实现用户交互

---

#### 1.3 DOM树

* DOM树是什么
  * 将 HTML 文档以树状结构直观的表现出来，我们称之为文档树或 DOM 树
  * 描述网页内容关系的名词
  * 作用:文档树直观的体现了标签与标签之间的关系

#### 1.4 DOM 对象

* DOM对象：浏览器根据html 标签生成的 JS 对象
  * 所有的标签属性都可以在这个对象上面找到
  * 修改这个对象的属性会自动映射到标签身上

* DOM的核心对象
  * 把网页内容当作对象来处理
* document 对象
  * 是 DOM 里提供的一个对象
  * 所以它提供的属性和方法都是用来访问和操作网页内容的
    * 例：document.write()
  * 网页中所有内容都在document里面

### 3. 操作元素内容

目标：能够修改元素的文本更换内容

* DOM 对象都是根据标签生成的，所以操作标签，本质上就是操作DOM对象
* 就是操作对象使用的点语法
* 如果想要更改标签里面的内容，可以使用以下方法：
* 学习路径：
  1. 对象.innerText 属性
  2. 对象.innerHTML 属性

#### 3.1对象.innerText 属性

* 将文本内容添加/更新到任意标签位置
* 显示纯文本，不解析标签

```
<body>
  <div class="box">我是文字的内容</div>
  <script>
    //1. 获取元素
    let box = document.querySelector('.box')
    //2.修改文字内容  对象.innerText
    console.log(box.innerText) //获取文字内容
    // box.innerText = '我是一个盒子'//修改文字内容
    box.innerText = '<strong>我是一个盒子</strong>'
    //文字没有加粗，说明innerText 不解析标签
  </script>
</body>
```

#### 3.2 对象.innerHTML 属性

* 将文本内容添加/更新到任意标签位置

* 会解析标签，多标签建议使用模板字符

* ```
  console.log(box.innerHTML)
  box.innerHTML='<h2>我要更换</h2>'
  ```

**两者的区别：**

1. innerText 只识别文本，不能解析标签
2. innerHTML能识别文本，能够解析标签

##### 2. 操作类名(className) 操作CSS

* 如果修改的样式比较多，直接通过 style 属性修改比较繁琐，我们可以通过借助于 css 类名的形式

* **语法：**

  ```
  //active 是一个css类名
  元素.className = 'active'
  ```

  ```
  <style>
      .nav {
        width: 200px;
        height: 200px;
        background-color: pink;
        color: red;
      }
      .box {
        width: 300px;
        height: 300px;
        background-color: skyblue;

        padding: 10px;
        border: 1px solid black;
      }
    </style>

      const div = document.querySelector('div')
      //2. 添加类名  class 是个关键字  我们用className
      div.className = 'box'
      //3. 如果原来已经有类名，则原来类名被覆盖
      div.className = 'nav box'//两个类名中的属性都保留
    </script>
  </body>
  ```

  

* **注意：**

  1. 由于class是关键字，所以使用className去替换
  2. className是使用新值换旧值，如果需要添加一个类，需保留之前的类名（`div.className = 'nav box'`）

##### 3.通过 classList 操作类控制CSS

### 5. 定时器—间歇函数

#### 5.1定时器函数介绍

* 网页中经常会需要一种功能：每隔一段时间需要自动执行一段代码，不需要手动去触发
* 例如：网页中的倒计时
* 要实现这种需求，需要定时器函数

#### 5.2 开启定时器

`setInterval(函数，间隔时间)`

* 作用：每隔一段时间调整这个函数
* 间隔时间单位是毫秒

```
 <script>
    //setInterval(函数，间隔时间)
    // function(){} 匿名函数
  setInterval(function(){
    console.log('一秒执行一次')
  },1000)

  function fn() {
    console.log('一秒钟执行两次')
  }
  //函数名不要加小括号
  setInterval(fn,1000)
  </script>
```

![image-20250421215111387](https://join-std.oss-cn-beijing.aliyuncs.com/2024011319/image-20250421215111387.png)

***注意：**

1. 函数名字==不需要加括号==
2. ==定时器返回的是一个id数字==

---

#### 5.3 关闭定时器

```
let 变量名 = setInterval(函数,间隔时间)
clearInterval(变量名)
```

```
  <script>
    //setInterval(函数，间隔时间)
    // function(){} 匿名函数
  let t = setInterval(function(){
    console.log('一秒执行一次')
  },1000)

  function fn() {
    console.log('一秒钟执行两次')
  }
  //函数名不要加小括号
  // setInterval(fn,1000)

  // 返回是一个id数字型
  let n = setInterval(fn,1000)
  //使用 let 定义变量 是因为定时器要开关，在这个过程中定时器的id 数值会改变，所以不使用const
  console.log(n)  //2
  console.log(t)  //1
  

  
// let 变量名 = setInterval(函数,间隔时间)
//   clearInterval(变量名)
//关闭定时器
clearInterval(n)  //关闭fn()
clearInterval(t)  //关闭匿名函数定时器
  
  </script>
```

* **定时器作用：可以根据时间自动重复执行某些代码**

* 开启定时器：`setInterval(函数,间隔时间)`
* 关闭定时器`let 变量名 = setInterval  clearInterval(变量名)`

## 1. 事件监听（绑定）

### 1.事件监听

目标：能够给DOM元素添加事件监听

* 什么是事件？
  * 事件是在==编程时系统内发生的动作或者发生的事情==
  * 比如用户在网页上==单击==一个按钮
* 什么是事件监听？
  * 就是让程序检测是否有事件产生，一旦有事件出发，就立即调用一个函数做出响应，也称为绑定事件或者注册事件
  * 比如鼠标经过显示下拉菜单，比如点击可以播放轮播图

---

#### 1.1事件监听

* 语法：

  `元素对象.addEventListener('事件类型',要执行的函数)`

* **事件监听三要素**

  * **事件源**：哪个dom元素被事件触发了，要获取dom元素
  * **事件类型：**用什么方式触发，比如鼠标单击 click、鼠标经过 mouseover 等
  * **事件调用的函数：**要做什么事

* **不是立即执行，当函数发生时才会执行**

* **注意：**

  1. 事件类型要加引号
  2. 函数是点击之后再执行，每次点击都会1执行一次

```
<body>
  <button>点击</button>
  <script>
    //点击按钮弹出一个对话框
    //1.事件源  按钮
    //1.1 获取元素
     //2. 事件类型  点击鼠标
    //3. 事件处理程序  弹出对话框
    const button = document.querySelector('button')
    button.addEventListener('click',function(){
      alert('你好~~')
    })
   
  </script>
</body>
```

* 事件源.on事件 = function(){}

* 事件源.addEventListener（事件，事件处理函数）

* 区别

  on方式会被覆盖，==addEventListener== 方式可以绑定多次，拥有事件更多特性，推荐使用

## 2. 事件流

### 1. 事件流与两个阶段说明

* 事件流指的是事件完整执行过程中的流动路径
  * 捕获阶段
  * 冒泡阶段

![image-20250422205627480](https://join-std.oss-cn-beijing.aliyuncs.com/2024011319/image-20250422205627480.png)

* 说明：假设页面里有个div，当触发事件时，会经历两个阶段，分别是捕获阶段、冒泡阶段
* 简单来说：捕获阶段是 从父到子，冒泡阶段是从子到父

---

### 2. 事件捕获

简单了解

* **事件捕获概念：**从父到子

  * 从DOM的根元素开始去执行对应的事件（从外到里）

* 事件捕获需要写对应的代码才能看到效果

* 代码：

  `DOM.addEventListener(事件类型，事件处理函数，是否使用捕获机制)`

  ```
  <body>
    <div class="father">
      <div class="son"></div>
    </div>
    <script>
  
      const fa = document.querySelector('.father')
      const son = document.querySelector('.son')
      document.addEventListener('click',function(){
        console.log('我是爷爷') 
        alert('我是爷爷')
      },true)
      fa.addEventListener('click',function(){
        console.log('我是爸爸') 
        alert('我是爸爸')
      },true)
      son.addEventListener('click',function(){
        console.log('我是儿子') 
        alert('我是儿子')
      },true)
      //爷爷  父亲  儿子
    </script>
  </body>
  ```

* 说明：
  * addEventListener第三个参数传入true代表是捕获阶段触发（很少使用）
  * 若传入false 代表冒泡阶段，==默认就是false==
  * 若是L0事件监听，则只有冒泡阶段，没有捕获

---

### 3. 事件冒泡

**事件冒泡概念：**从子到父

当一个元素的事件被触发时，同样的事件将会在该元素的所有祖先元素中依次被触发。这一过程被称为事件冒泡

* 简单理解：当一个元素触发事件后，会依次向上调用所有父级元素的==同名事件==//同一种类型的事件
* 事件冒泡是默认存在的
* L2事件监听第三个参数是false，或者默认都是冒泡

```
const fa = document.querySelector('.father')
    const son = document.querySelector('.son')
    document.addEventListener('click',function(){
      console.log('我是爷爷') 
      alert('我是爷爷')
    })
    fa.addEventListener('click',function(){
      console.log('我是爸爸') 
      alert('我是爸爸')
    })
    son.addEventListener('click',function(){
      console.log('我是儿子') 
      alert('我是儿子')
    })
    //儿子  爸爸 爷爷
```

---

### 4. 阻止冒泡

* **问题**:因为默认就有冒泡模式的存在，所以容易导致事件影响到父级元素

* **需求**:若想把事件就限制在当前元素内，就需要阻止事件冒泡

* **前提**:阻止事件冒泡需要拿到事件对象

* 语法：

  `事件对象.stopPropagation()`//方法

* 注意：此方法可以阻断事件流通传播，==不光在冒泡阶段有效，捕获阶段也有效==

```
son.addEventListener('click',function(e){
//用到了事件对象
  console.log('我是儿子') 
  alert('我是儿子')
  e.stopPropagation()
})
```

### 5. 解绑事件

on 事件方式，直接使用null 覆盖就可以实现事件的解绑

**语法**：

##### L0解绑

```
//绑定事件
btn.onclick = function(){
  alert('点击了')  
}
//解绑事件
btn.onclick = null
```

```
<script>
    const btn = document.querySelector('button')

    //绑定事件
    btn.onclick = function(){
      alert('点击了')  
    }
    //解绑事件
    btn.onclick = null
  </script>
```

##### L2 解绑

addEventListener 方式，必须使用：

`removeEventListener(事件类型，事件处理函数，[获取捕获或者冒泡阶段])`

例如：

```
function fn(){
  alert('点击了')
}
//绑定事件
button.addEventListener('click',fn)//回调函数，只需要写函数名
button.removeEventListener('click',fn)
```

**注意：匿名函数无法被解绑**



##### 两种注册事件的区别

* 传统on 注册（L0）
  * 同一个对象,后面注册的事件会覆盖前面注册(同一个事件)
  * 直接使用nul覆盖偶就可以实现事件的解绑
  * 都是冒泡阶段执行的
* 事件监听注册（L2）
  * 语法:addEventListener(事件类型,事件处理函数,是否使用捕获)
  * 后面注册的事件不会覆盖前面注册的事件(同一个事件)
  * 可以通过第三个参数去确定是在冒泡或者捕获阶段执行
  * 必须使用removeEventListener(事件类型,事件处理函数,获取捕获或者冒泡阶段)
  * 匿名函数无法被解绑

## 3. 事件委托

事件委托是利用事件流的特征解决一些开发需求的知识技巧

* 优点：==减少注册次数，可以提高程序性能==
* 原理：事件委托是利用事件冒泡的特点
  * 给==父元素==注册事件，当我们触发子元素的时候，会冒泡到父元素身上，从而触发父元素的事件

* 找到真正的触发元素：`事件对象.target.tagName = 'LI'(注意大写)`

```
<body>
  <ul>
    <li>第1个盒子</li>
    <li>第2个盒子</li>
    <li>第3个盒子</li>
    <li>第4个盒子</li>
    <li>第5个盒子</li>
    <p>我不需要变色</p>
  </ul>
  <script>
    //点击每一个小li  当前li文字变成红色
    //事件委托方式
    //1. 获取父元素
    const fa = document.querySelector('ul')
    fa.addEventListener('click',function(e){
      // this .style.color = 'red'
      //得到事件对象
      // console.log(e.target)//就是我们点击的对象
      // e.target.style.color = 'red'
      //我们只要点击li才会有效果
      if(e.target.tagName === 'LI'){
        e.target.style.color = 'red'
      }
    })

  </script>
</body>
```

### 5.3 时间戳的使用

  * 是指1970年01月01日00时00分00秒起至现在的毫秒数它是一种特殊的计量时间的方式

* 算法:

  * 将来的时间戳 -现在的时间戳 = 剩余时间毫秒数

  * 剩余时间毫秒数 转换为 剩余时间的 年月日时分秒 就是 倒计时时间

    比如 将来时间戳 2000ms-现在时间戳1000ms=1000ms

  * 1000ms转换为就是 0小时0分1秒

   * 但是只能得到当前的时间戳，而前面两种可以返回指定时间的时间戳

### 7.2.JS的执行机制

* 渲染引擎
* 解析JS引擎

JavaScript 语言的一大特点就是==单线程==，也就是说，==同一个时间只能做一件事。==

单线程就意味着，所有任务需要排队，前一个任务结束，才会执行后一个任务。这样所导致的问题是:如果 JS 执行的时间过长，这样就会造成页面的渲染不连贯，导致页面渲染加载阻塞的感觉。

* 为了解决这个问题，利用==多核CPU 的计算能力==，HTML5允许JavaScript 脚本创建多个线程。于是，==JS出现了同步和异步==

* **本质区别**：一条流水线上各个流程的执行顺序不同

#### 1.同步任务

同步任务都在主线程上执行，形成一个执行栈

#### 2. 异步任务

* JS的异步是通过回调函数实现的
* 一般而言，异步任务有以下三个类型

1. 普通时间，如click、resize 等
2. 资源加载，如 load、error等
3. 定时器，包括 setInterval、 setTimeout 等

异步任务相关添加到任务对列中（任务队列也称为消息队列）

![image-20250424144335629](https://join-std.oss-cn-beijing.aliyuncs.com/2024011319/image-20250424144335629.png)

**同步任务是立马能执行的**，**异步任务是比较耗时的**

#### 3. JS的执行机制

1. 先执行栈找中的同步任务
2. 异步任务放入任务队列中
3. 一旦执行栈中所有同步任务执行完毕，系统就会按次序读取任务队列中的异步任务，于是被读取的异步任务结束等待状态，进入执行栈，开始执行
4. ==这样的过程称为事件循环==

![image-20250424151656451](https://join-std.oss-cn-beijing.aliyuncs.com/2024011319/image-20250424151656451.png)

由于主线程不断的重复获得任务、执行任务、再获取任务、再执行，所以这种机制被称为==事件循环（event loop）==

### 9.2 箭头函数this

普通函数this指向

```javascript
//以前this的指向：谁调用的这个函数，this就指向谁
    console.log(this)//window
    //普通函数
    function fn(){
      console.log(this)
    }
    window.fn()//window
    
    const obj = {
      name:'andy',
      sayHi: function () {
        console.log(this)
      }
    }
    obj.sayHi()//obj
```

箭头函数this

在箭头函数出现之前，每个新函数根据它时如何被调用的来定义这个函数的this值，非常麻烦

箭头函数不会创建自己的this，它只会从自己的作用域链的上一层沿用this

```javascript
const obj = {
	uname: uname,
	sayHi: () => {
		console.log(this)
	}
}
obj.sayHi()//window
```

这里this的上一层作用域是obj，obj的this指向window,箭头函数的this沿用obj的this--->window

**在开发中【使用箭头函数前需要考虑函数中的this的值】，事件回调函数使用箭头函数时，this为全局的window，因此DOM事件回调函数为了简便，不推荐使用箭头函数**
