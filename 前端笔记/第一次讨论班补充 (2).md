### iconfont是如何实现字体图标的？



#### 一、核心原理

Iconfont 字体图标 = **把图标做成「文字字体」**

浏览器把图标当成**汉字 / 字母**渲染，本质是：

> 每个图标 = 字体里的一个「特殊字符 + 字形」

#### 

设计师把图标（箭头、搜索、菜单、首页…）

做成矢量图形，导入字体工具，映射到**字体编码（Unicode）**

1. iconfont 就是**把图标做成字体文件**

2. 每个图标对应一个 **Unicode 字符编码**

3. CSS 引入自定义字体，标签绑定字体，输入编码就出图标

4. 改色用 color、改大小用 font-size、矢量不失真



##### 字体编码 = 给每个图标 / 文字分配一个唯一「身份证编号」

### meta标签分三类

1. **charset 编码类**：规定网页文字编码（防止乱码）
2. **name + content 功能类**：配置各种网页属性
3. **http-equiv 响应头类**：模拟服务器 HTTP 响应头

```
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    //1. 统一汉字、符号编码，杜绝中文乱码
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    //2. 移动端视口适配（做手机网页必加）
	属性解释：
	width=device-width：宽度等于手机屏幕宽度
	initial-scale=1.0：默认不放大、不缩小
	不加这个，手机端页面会缩小、排版乱
    <title>Document</title>
    <style>
      /* 给 label 加点 hover 反馈，让用户知道可以点 */
      label[for] {
        cursor: pointer;
        user-select: none;
        transition: color 0.2s;
      }
      label[for]:hover {
        color: #1890ff;
      }
    </style>
  </head>
  <body>
    <!-- 正确示范 -->
    <label for="userName">用户名</label>
    <input id="userName" name="userName" type="text" placeholder="请输入昵称" />
  </body>
</html>

```

1. 字符编码型**charset 类**：设置编码，防乱码

   语法：`<meta charset="值">`

2. name + content（功能说明 / SEO / 适配）

   - `name`：给当前 meta 标签**起名字 / 归类**，告诉浏览器这是「视口配置标签」

   - `content`：是**具体配置参数区**，把一堆规则写在这里

   - `name`：定义这个 meta 是**干嘛的**

     `content`：写**具体生效的规则**

     ![image-20260406184120168](https://join-std.oss-cn-beijing.aliyuncs.com/2024011319/image-20260406184120168.png)

3. http-equiv + content（模拟服务器响应头）：控制缓存、跳转、模拟响应头



**`http-equiv`：模拟后端 HTTP 响应头，控制缓存、跳转、兼容**用来在前端 HTML 里，模拟「后端服务器响应头」，控制浏览器行为。

1. 网页自动刷新 / 自动跳转

   ```
   <meta http-equiv="refresh" content="5;url=login.html">
   ```

   意思：**5 秒后自动跳到登录页**

2. 控制浏览器缓存（开发 / 测试高频）

3. 强制 IE 用最新内核渲染（兼容旧项目）

### Form表单使用

自带**回车自动聚焦 / 提交**原生交互

- 表单内按回车，会触发默认提交行为、联动焦点跳转；
- 不套 form：回车没任何反应，用户填表体验割裂。

##### 2. 原生表单校验自带生效（HTML5 校验）

`required` / `pattern` / `type="email/tel/number"` 这些：

- 套 form：点击表单内 `<button type="submit">`，**自动触发原生弹窗校验、拦截提交**
- 不套 form：原生 H5 校验**直接失效**，只能自己手写 JS 判断

#####  3. 一键重置超级方便

套 form：

- 加 `<button type="reset">重置</button>`，一键清空所有输入框、单选、复选
- 不套 form：自己写 JS 逐个清空 value

##### 4. 原生快速获取整组表单数据

DOM 操作：

- `form元素.elements` 直接拿到**所有 input/select/textarea 集合**
- 搭配 `name` 属性，一键拿整表数据，不用逐个 `document.querySelector`

#### 不套`<form>`优点（极少）

1. 结构随意，不用管 form 默认行为拦截
2. 超简单零散输入（1~2 个框），代码更轻

#### 致命缺点

1 H5 原生校验全部废了，全靠手写 JS 判断

2 不能用 reset 一键重置，清空麻烦

3 回车填表交互烂，用户体验差

4 获取所有表单值要逐个拿，代码冗余

5 语义差，无障碍、爬虫识别不出来是完整报名表

6 后期加校验、联动、必填项维护成本翻倍
