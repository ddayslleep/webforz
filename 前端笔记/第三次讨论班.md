# 一、常用其他基础属性

## 1. 透明度（opacity）

核心作用：控制元素的透明程度，影响元素本身及内部所有内容（文字、图片、子元素等）。

取值范围：0 ~ 1（无单位），取值越接近0，透明度越高；越接近1，透明度越低。

易混点：

- `opacity`：作用于整个元素（包括子元素），所有内容一起变透明，无法单独控制子元素透明度。
- `rgba()`：仅作用于元素的背景色/边框色，不影响元素内部的文字、子元素，可单独控制背景透明。

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>opacity与rgba</title>
    <style>
        body {
            background: #f0f0f0; 
        }
        .parent-opacity {
            width: 200px;
            height: 150px;
            background: #ff3333; 
            opacity: 0.6;
            margin: 20px;
            padding: 10px;
            font-weight: bold;
        }
        .child-opacity {
            width: 100px;
            height: 70px;
            background: #0066ff; 
            opacity: 0.7; 
            color: #fff;
            font-weight: bold;
        }

        .parent-rgba {
            width: 200px;
            height: 150px;
            background: rgba(255, 51, 51, 0.6); 
            margin: 20px;
            padding: 10px;
            font-weight: bold;
        }
        .child-rgba {
            width: 100px;
            height: 70px;
            background: #0066ff; 
            color: #fff;
            font-weight: bold;
        }
    </style>
</head>
<body>
    <div class="parent-opacity">
        父元素
        <div class="child-opacity">子元素</div>
    </div>

    <div class="parent-rgba">
        父元素
        <div class="child-rgba">子元素</div>
    </div>
</body>
</html>
```

## 2. 光标类型（cursor）

核心作用：控制鼠标悬浮在元素上时，光标的显示样式，提升用户交互体验。

常用取值：

- cursor: default; → 默认光标（箭头），大多数元素的默认值。
- cursor: pointer; → 手型光标，常用于按钮、链接（a标签默认此样式），提示用户“可点击”。
- cursor: text; → 文本光标（竖线），常用于输入框（`input、textarea`默认此样式），提示用户“可输入”。
- cursor: move; → 移动光标（十字箭头），常用于可拖动的元素（如弹窗、滑块）。
- cursor: not-allowed; → 禁止光标（带斜杠的圆圈），常用于不可点击的元素（如禁用按钮）。

注意事项：光标样式仅在鼠标“悬浮”在元素上时生效，若元素设置了pointer-events: none，光标样式会失效

# 三、平面转换

核心概念：transform属性用于实现元素在“平面内”的形态变化，包括位移、旋转、缩放、倾斜四种基础效果，可单独使用，也可组合使用，核心是“不改变元素的文档流位置”（元素仍占据原来的空间）。

注意：平面转换仅作用于元素的视觉形态，不改变元素的尺寸、位置（文档流中），也不影响其他元素的布局。

## 1. 平移（translate）

核心作用：让元素在平面内（X轴、Y轴）移动，不改变元素的尺寸和形状。

### 基础语法

- transform: translateX(值); → 沿X轴（水平方向）移动，正值向右，负值向左。
- transform: translateY(值); → 沿Y轴（垂直方向）移动，正值向下，负值向上。
- transform: translate(X值, Y值); → 同时沿X轴、Y轴移动，两个值用逗号分隔。

### 取值规则（重点）

- 像素值（`px`）：固定距离移动，如`translateX(50px)` → 向右移动50px。

- 百分比（`%`）：参照“元素自身的尺寸”计算移动距离（重点！易混点），如`translateX(50%)` → 向右移动元素自身宽度的50%，`translateY(50%)` → 向下移动元素自身高度的50%。

  

- 易混点：translate的百分比是“相对于自身”，而margin的百分比是“相对于父级”，比如子级`margin-left: 50%;` 是相对于父级宽度的50%，而`translateX(50%)`是相对于自身宽度的50%。

## 2. 旋转（rotate）

核心作用：让元素绕“转换原点”旋转，改变元素的角度，不改变元素的尺寸和位置。

### 基础语法

`transform: rotate(角度值);` → 角度单位为deg（度），正值顺时针旋转，负值逆时针旋转，如rotate(30deg) → 顺时针旋转30度，`rotate(-30deg)` → 逆时针旋转30度。

### 关键知识点：转换原点（transform-origin）

转换原点：元素旋转、缩放、倾斜的基准点，默认值是“元素的中心点”（center center）。

#### 语法与取值

transform-origin: 水平原点位置 垂直原点位置;（两个值，分别控制水平和垂直方向）

- 方位名词：`left（左）、top（上）、right（右）、bottom（下）、center（中）`
- 像素值（`px`）：精确设置原点位置，如`transform-origin: 50px 50px;` → 以元素左上角为起点，向右50px、向下50px的位置为旋转原点。
- 百分比（`%`）：参照元素自身尺寸，如`transform-origin: 100% 0%`; → 右上角。

### 多重转换（易混点）

当需要同时实现“平移+旋转”“旋转+缩放”等多种效果时，需在一个transform属性中写多个转换函数，用空格分隔，**顺序至关重要**。

- 正确顺序：先平移，再旋转（`transform: translate(800px) rotate(360deg);`）。
- 错误顺序：先旋转，再平移（`transform: rotate(360deg) translate(800px);`）→ 旋转会改变元素的坐标轴方向，导致平移方向错乱（比如原本向右平移，旋转后可能变成向上/向下平移）。
- 注意：不可拆分写多个transform属性，会发生样式层叠，只有最后一个transform生效。

## 3. 缩放（scale）

核心作用：放大或缩小元素，改变元素的尺寸，不改变元素的文档流位置。

### 基础语法

- `transform: scaleX(倍数);` → 沿X轴（水平方向）缩放，倍数大于1放大，小于1缩小，等于1不变。
- `transform: scaleY(倍数);` → 沿Y轴（垂直方向）缩放，规则同上。
- `transform: scale(倍数1, 倍数2);` → 同时沿X轴、Y轴缩放，两个倍数用逗号分隔。
- `transform: scale(倍数);` → 简写形式，同时缩放X轴和Y轴，如scale(1.5) → 整体放大1.5倍。

### 注意事项

- 缩放倍数无单位，仅为数字，如scale(2) 而非 scale(2px)。
- 缩放会影响元素内部的所有内容（文字、图片等），放大后可能会出现模糊，建议缩放倍数不要过大。
- 缩放原点与旋转原点一致，默认是元素中心点，可通过transform-origin修改，如scale(2)配合transform-origin: left top; → 以左上角为原点放大。

## 4. 倾斜（skew）

核心作用：让元素沿X轴、Y轴倾斜，改变元素的形状（类似“拉伸”），不改变元素的尺寸和文档流位置。

### 基础语法

- `transform: skew(角度1);` → 仅沿X轴倾斜，角度单位为deg，正值向右倾斜，负值向左倾斜。
- `transform: skew(角度1, 角度2);` → 同时沿X轴、Y轴倾斜，两个角度用逗号分隔，角度2控制Y轴倾斜（正值向下，负值向上）。

注意事项：倾斜角度不宜过大（建议0~45deg），角度过大会导致元素严重变形，影响视觉效果；倾斜仅改变元素形状，不改变元素的位置和尺寸。

## 5. 渐变（background-image）

核心作用：给元素背景添加“渐变色”，替代单一背景色，提升页面质感，分为线性渐变和径向渐变两种，语法均需写在background-image属性中。

### （1）线性渐变（linear-gradient）

核心：从一个颜色平滑过渡到另一个/多个颜色，呈线性分布。

#### 基础语法

`background-image: linear-gradient`(渐变方向, 颜色1 终点位置, 颜色2 终点位置, ...); 

- 渐变方向（可选）：两种取值方式，二选一即可。        
  - 方位名词：to left（向左）、to right（向右）、to top（向上）、to bottom（向下，默认值）、to top right（向右上）等。
  - 角度度数：如45deg（顺时针45度）、135deg等，角度从X轴正方向开始计算，顺时针为正。
- 颜色与终点位置（可选）：
  - 颜色：可写多个颜色，用逗号分隔，如red, blue → 从红色过渡到蓝色。
  - 终点位置：用百分比表示，控制颜色过渡的结束位置，如red 30%, blue 70% → 红色在30%位置结束，蓝色从70%位置开始，中间是过渡色；不写终点位置，颜色会平均分布。

### （2）径向渐变（radial-gradient）

核心：从一个中心点向外扩散，颜色呈圆形/椭圆形渐变，常用于给按钮、卡片添加高光效果。

#### 基础语法

`background-image: radial-gradient`(半径 at 圆心位置, 颜色1 终点位置, 颜色2 终点位置, ...);

- 半径（可选）
- 圆心位置（可选）：控制渐变的中心点，取值为像素值、百分比或方位名词。
- 颜色与终点位置：同线性渐变，可写多个颜色，终点位置用百分比控制。

易混点：线性渐变是“线性分布”，径向渐变是“扩散分布”；两者均需写在background-image中，不能直接写在background中（可与background-color叠加）。

# 四、空间转换

核心概念：在平面转换的基础上，增加了“Z轴”（垂直于屏幕的轴），实现元素的“立体效果”，核心是让元素在3D空间中变化，需配合透视效果才能看到明显的立体视觉。

注意：空间转换需要开启“3D空间”，否则元素仍会处于平面中，无法呈现立体效果。

## 1. 空间平移（translate3d）

核心作用：让元素在3D空间中（X轴、Y轴、Z轴）移动，Z轴移动决定元素的“前后位置”（靠近/远离屏幕）。

### 基础语法

- `transform: translate3d(x, y, z);` → 同时沿X、Y、Z轴移动，三个值必须全部设置（缺一不可），取值为像素值或百分比（参照自身尺寸）。
- 单独轴平移：`transform: translateX(x);、transform: translateY(y);、transform: translateZ(z);` 

### 关键细节

- Z轴取值：正值 → 元素向屏幕方向移动（靠近用户），负值 → 元素向屏幕后方移动（远离用户）。
- translateZ(z) 无法单独看到效果，必须配合“透视效果”（perspective），否则Z轴移动会被平面化，无法呈现前后差异。
- 易混点：平面平移（translate）只有X、Y轴，空间平移（translate3d）多了Z轴，且三个值必须写全。

## 2. 透视效果（perspective）

核心作用：模拟人眼观察物体的“近大远小、近实远虚”效果，让空间转换的立体效果更明显，是实现3D视觉的关键。

### 基础语法与取值

`perspective: 数值px;` → 数值越大，透视距离越远（立体效果越弱）；数值越小，透视距离越近（立体效果越强）。

常用取值：800px ~ 1200px（兼顾立体效果和视觉舒适度，最常用）。

### 使用注意事项（重点！）

- perspective 需添加给“元素的直接父级”，而非元素本身，这样所有子元素都会继承透视效果，便于统一控制。
- 没有perspective，空间转换的效果会不明显，甚至和平面转换无区别。
- 透视效果仅影响“子元素”的空间显示，不影响父元素本身的样式。

## 3. 空间旋转（rotate3d）

核心作用：让元素绕3D空间中的轴（X、Y、Z轴）旋转，实现立体旋转效果，比平面旋转更具立体感。

### 基础语法

- 单独轴旋转：`transform: rotateX(角度);、transform: rotateY(角度);、transform: rotateZ(角度);` → 角度单位为deg，正值、负值的判断遵循“左手法则”。
- 自定义旋转轴：`transform: rotate3d(x, y, z, 角度);` → x、y、z取值为0~1之间的数字，用于定义旋转轴的方向（0表示不沿该轴旋转，1表示沿该轴旋转），角度表示旋转角度。

### 左手法则（判断旋转方向，重点！）

用左手握住旋转轴，拇指指向轴的正方向，其余四个手指弯曲的方向，就是旋转的“正值方向”：

- `rotateX(deg)`：拇指向上（X轴正方向），手指弯曲方向为顺时针旋转（正值）。
- `rotateY(deg)`：拇指向右（Y轴正方向），手指弯曲方向为顺时针旋转（正值）。
- `rotateZ(deg)`：拇指垂直屏幕向外（Z轴正方向），手指弯曲方向为顺时针旋转（正值）。

实用示例：`rotateX(60deg)` → 元素绕X轴顺时针旋转60度，呈现“向前倾倒”的效果；

`rotateY(90deg)` → 元素绕Y轴顺时针旋转90度，呈现“侧面朝向用户”的效果。

## 4. 立体呈现（transform-style）

核心作用：设置元素的子元素是“处于3D空间中”还是“处于平面中”，是实现立体图形（如正方体）的核心属性。

### 基础语法与取值

- `transform-style: flat;` → 默认值，子元素处于平面中，无法呈现3D效果（即使父元素设置了perspective）。
- `transform-style: preserve-3d;` → 关键值，子元素处于3D空间中，能够继承父元素的3D效果，实现立体呈现。

### 实现立体图形的步骤（重点）

1. 给父元素添加`transform-style: preserve-3d;` → 开启3D空间，让子元素处于3D环境中。
2. 父元素设置position: relative;，子元素设置position: absolute; → 让子元素可以自由定位（叠加在一起）。
3. 调整子元素的位置：通过`translate3d（或translateZ）、rotateX/rotateY`等空间转换，让子元素分别处于父元素的不同面（如前面、后面、左面、右面）。

注意事项：`transform-style: preserve-3d;` 必须添加给“父元素”，子元素才能呈现3D效果；若父元素设置了`overflow: hidden;`，可能会裁剪3D效果，导致立体图形显示异常。

## 5. 空间缩放（scale3d）

核心作用：在3D空间中放大或缩小元素，与平面缩放类似，增加了Z轴的缩放控制。

### 基础语法

- `transform: scale3d(x, y, z);` → 同时沿X、Y、Z轴缩放，x、y、z分别是三个轴的缩放倍数，无单位，大于1放大，小于1缩小。
- 单独轴缩放：`transform: scaleX(倍数);、transform: scaleY(倍数);、transform: scaleZ(倍数);` → 与平面缩放语法一致。

注意：`scaleZ(倍数)` 单独使用效果不明显，通常配合其他空间转换（如`rotateX、translateZ`），才能看到3D缩放效果；日常开发中，平面缩放（scale）已能满足大部分需求，空间缩放使用较少。

# 五、动画

核心概念：动画用于实现元素“多状态”的连续变化，过程可控（可设置时长、延迟、重复次数等），区别于过渡（仅能实现两个状态的变化）。

核心组成：动画由“定义动画（@keyframes）”和“使用动画（animation）”两部分组成，缺一不可。

## 1. 动画基础：定义动画（@keyframes）

作用：定义动画的“关键帧”，即动画的起始状态、中间状态、结束状态，控制动画的变化过程。

### 两种定义方式

- 方式1：from → to（适合两个状态的动画）        

  ```css
  @keyframes 动画名称{ 
  from { /* 起始状态：动画开始时的样式 */ } 
  to { /* 结束状态：动画结束时的样式 */ } 
  }
  ```

  

- 方式2：百分比（适合多状态的动画，更灵活） 

  ```css
  @keyframes 动画名称 { 
  0% { /* 起始状态（动画开始时） */ } 
  20% { /* 中间状态（动画进行到20%时） */ }  
  50% { /* 中间状态（动画进行到50%时） */ } 
  100% { /* 结束状态（动画结束时） */ } 
  }
  ```

  

### 注意事项

- 动画名称：自定义名称（如change），命名要规范，避免与CSS关键字冲突。
- 关键帧样式：可设置任意CSS属性（如`width、height、transform、background-color`等），实现多维度的变化。
- 0%和100%是必须的（from等价于0%，to等价于100%）

## 2. 动画使用：复合属性（animation）

作用：将动画的各个属性（时长、延迟、重复次数等）整合在一起，简化代码，核心语法是“复合属性”，也可拆分单独设置。

### 复合属性语法

animation: 动画名称 动画时长 速度曲线 延迟时间 重复次数 动画方向 执行完毕时状态;

关键说明（重点记，易混点多）：

- 必选属性（缺一不可）：动画名称、动画时长（两个属性必须赋值，否则动画无法生效）。
- 取值顺序：无严格顺序，但两个时间值需注意——第一个时间是“动画时长”，第二个时间是“延迟时间”（若只有一个时间值，默认是动画时长）。

### 易混点与实用细节

- 动画时长 vs 延迟时间：两个都是时间值，第一个是时长，第二个是延迟，如animation: change 2s 1s; → 动画时长2秒，延迟1秒开始执行。
- `forwards` vs `backwards：forwards`是“停在最后一帧”，backwards是“回到第一帧”，开发中若想让动画结束后保持最终效果，必须设置forwards。
- 逐帧动画：设置`animation-timing-function: steps(数字)`;，数字表示将动画分成多少帧，适合实现“帧动画”（如精灵图动画），区别于匀速/慢快慢的平滑动画。
- 动画暂停：常用场景——鼠标悬浮时暂停动画，如`.box:hover { animation-play-state: paused; }`，鼠标离开后恢复播放。

# 六、移动端适配方案

核心需求：移动端设备屏幕尺寸多样（如320px、375px、414px等），需让网页在不同尺寸的设备上正常显示，不出现拉伸、错位、溢出等问题，核心是“自适应”。

常用适配方案：宽度自适应（百分比、Flex）、等比适配（rem、vw），重点讲解rem和vw适配（最常用、最实用）。

## 1. 基础适配方案概述

- 宽度适配：让网页宽度随设备视口宽度变化，避免固定宽度导致的溢出，常用方式：
  - 百分比布局：元素宽度用百分比表示（如`width: 100%;`），相对于父级宽度自适应。
  - Flex布局：通过`display: flex;` 让子元素自适应父级宽度，灵活分配空间（最常用）。
- 等比适配：让网页元素的宽高比随视口尺寸变化，避免元素变形，常用方式：`rem、vw`（重点）。

## 2. rem适配（重点）

### 核心概念

rem是“相对单位”，其大小是“相对于HTML标签的字号（font-size）”计算的，公式：1rem = 1个HTML标签的font-size值。

例：若HTML标签设置font-size: 37.5px;，则1rem = 37.5px，元素width: 10rem; → 实际宽度为375px。

### 核心原理

通过“改变HTML标签的font-size值”，让rem对应的实际像素值随视口宽度变化，从而实现元素尺寸的等比适配——视口越宽，HTML字号越大，rem对应的像素值越大，元素越大；反之则越小。

### 使用步骤

1. 给HTML标签设置基础font-size（默认值，如font-size: 37.5px;，对应375px视口宽度）。
2. 页面中所有元素的尺寸（`width、height、padding、margin`等），均用rem表示（像素值 ÷ HTML字号 = rem值）。
3. 通过媒体查询或JS，根据不同视口宽度，修改HTML标签的font-size值，实现多设备适配。

### 两种实现方式

- 方式1：媒体查询 + rem  

  ```css
   /* 视口宽度375px时，HTML字号37.5px */ 
  @media (width: 375px) {
  html {
  font-size: 37.5px; 
  }
  } 
  /* 视口宽度320px时，HTML字号32px */ 
  @media (width: 320px) { 
  html {
  font-size: 32px; 
  } 
  }
  /* 元素尺寸用rem表示 */ 
  .box { 
  width: 5rem; 
  /* 375px视口下，5×37.5=187.5px */ 
  height: 3rem; 
  /* 375px视口下，3×37.5=112.5px */ 
  }
  ```

  

- 方式2：JS + rem

  使用flexible.js（专门用于移动端适配的JS库），自动检测视口宽度，计算并设置HTML标签的font-size值，无需手动写媒体查询。

  使用方法：在HTML页面底部引入flexible.js文件即可（script标签引入），引入后，HTML字号会自动随视口宽度变化。

### 易混点与注意事项

- rem是“相对于HTML字号”，不是相对于父级，区别于`em`（`em`相对于父级字号），避免混淆。
- HTML字号的设置：通常以“视口宽度的1/10”为标准（如375px视口 → 37.5px，414px视口 → 41.4px），方便计算。
- 像素转rem：计算公式为“像素值 ÷ HTML字号 = rem值”，如100px在HTML字号37.5px时，对应100 ÷ 37.5 ≈ 2.67rem。

## 3. 媒体查询

核心作用：检测视口的尺寸（宽度、高度等），当满足某个条件时，执行对应的CSS样式，实现“差异化适配”（如不同视口显示不同布局、不同样式）。

### 基础语法

```css
@media (媒体特性) {
  选择器 {
    CSS属性; /* 满足条件时执行的样式 */
  }
}
```

### 常用媒体特性（重点）

- width: 视口宽度，如`@media (width: 375px)` → 视口宽度等于375px时生效。
- min-width: 最小视口宽度，如`@media (min-width: 375px)` → 视口宽度大于等于375px时生效（常用）。
- max-width: 最大视口宽度，如`@media (max-width: 414px)` → 视口宽度小于等于414px时生效（常用）。

实用场景：配合rem适配，设置不同视口的HTML字号；或在小屏幕设备上隐藏某些元素、调整布局（如从多列布局改为单列布局）。

## 4. vw适配方案

### 核心概念

vw、vh是“相对视口的单位”，尺寸相对于“视口的宽度/高度”计算，无需依赖HTML字号，更简洁。

- `vw：viewport width（视口宽度）`，1vw = 视口宽度的1/100。
- `vh：viewport height（视口高度）`，1vh = 视口高度的1/100。

例：视口宽度375px时，1vw = 3.75px，元素width: 50vw; → 实际宽度为375 × 50% = 187.5px。

### 使用注意事项（易混点）

- 开发中，vw和vh**不要混用**！因为全面屏设备的视口高度尺寸较大，混用会导致元素宽高比失调、变形（如原本正方形的盒子，混用后变成长方形）。
- 优先使用`vw`：日常适配中，主要关注“宽度适配”，用`vw`控制元素的宽度、padding、margin等，高度可根据宽高比用`vw`计算，或固定像素（避免变形）。
- 优势：无需设置HTML字号，无需媒体查询/JS辅助，直接根据视口宽度自适应，代码更简洁；劣势：部分旧版浏览器兼容性稍差（现代浏览器均支持）。

# 七、Less预处理

核心概念：Less是CSS预处理器，在CSS的基础上增加了变量、嵌套、运算、导入等功能，让CSS代码更具逻辑性、可维护性，便于修改和复用，最终会编译成普通CSS文件，供浏览器识别。

注意：浏览器无法直接识别Less代码，必须通过编译工具（如VS Code的Easy Less插件），将Less文件编译成CSS文件，才能在网页中使用。

## 1. 注释

Less支持两种注释方式，区别于普通CSS注释，且编译后表现不同：

- 单行注释：语法// 注释内容，快捷键Ctrl+/，**编译后不会保留在CSS文件中**，适合写开发备注（如“此处是导航样式”）。
- 块注释：语法/* 注释内容 */，快捷键Shift+Alt+A，**编译后会保留在CSS文件中**，适合写公共说明（如“版权声明”）。

## 2. 运算

Less支持加减乘除运算，可直接在样式中计算数值，无需手动计算，减少错误，便于适配（如rem转像素、宽高比计算）。

### 语法规则

- 加减乘除：直接写表达式，除法需要“加括号”或“加点”（必须，否则编译报错）。
- 单位规则：表达式中有多个单位时，最终CSS中以“第一个单位”为准。

实用示例：

```less
// 正确写法（除法）
width: (68 / 37.5rem); // 括号写法
width: 68./37.5rem;    // 加点写法

// 加减运算（单位一致）
height: 100px + 50px; // 结果150px
padding: 20px - 5px;  // 结果15px

// 单位不一致（以第一个单位为准）
width: 100px + 5rem;  // 编译后为105px（不推荐，尽量保持单位一致）
```

## 3. 嵌套

核心作用：快速生成后代选择器，简化代码结构，让代码更清晰（避免写冗长的后代选择器），尤其适合复杂的嵌套布局。

### 基础语法

```less
.父级选择器 {
  // 父级样式
  .子级选择器 {
    // 子级样式（后代选择器：.父级选择器 .子级选择器）
    .孙级选择器 {
      // 孙级样式（.父级选择器 .子级选择器 .孙级选择器）
    }
  }
}
```

### 关键知识点：&符号（重点！）

&符号用于“指代父选择器”，避免生成后代选择器，常用于伪类（如:hover、:active）、伪元素（如::before、::after）。

```less
.btn {
  color: #fff;
  // & 指代.btn，编译后为.btn:hover
  &:hover {
    color: #ff0;
  }
  // 编译后为.btn::before
  &::before {
    content: "";
  }
}
```

易混点：没有&符号，会生成后代选择器；有&符号，会直接拼接父选择器，不会生成后代。

## 4. 变量（核心优势）

核心作用：存储常用数据（如颜色、字号、间距等），方便统一修改和复用，避免重复写相同数值，提升开发效率。

### 语法规则

- 定义变量：@变量名: 数据;（变量名可自定义，建议规范命名，如`@primary-color、@base-font-size`，便于识别和维护）
- 使用变量：直接在需要的地方引用变量名，如`color: @primary-color;`，Less编译时会自动将变量替换为对应的数据。

实用示例：

```less
// 定义变量（存储主题色、基础字号）
@primary-color: #0066cc; // 主题色
@base-font-size: 16px;   // 基础字号

// 使用变量
.header {
  background-color: @primary-color;
  font-size: @base-font-size;
}
.nav {
  color: @primary-color;
  font-size: @base-font-size + 2px; // 配合运算使用
}
```

注意事项：

变量定义后可全局使用，若局部重新定义变量，会覆盖全局变量（局部优先级更高）；

变量名区分大小写.

##### **`perspective`** 

用于**定义透视投影矩阵**，建立三维渲染的观察空间，其核心作用是为**当前元素的所有子代三维变换元素提供统一的观察视点与景深**。

根据 CSS Transform Module 规范：

- `perspective` 作用于**父级容器**，用于创建一个**共享的 3D 渲染上下文**
- 所有应用了 3D transform 的子元素，都会在这个透视空间内进行渲染
- 在此案例中

- `.player` 是**执行三维旋转的元素**
- `.player-container` 是其**直接父容器**

因此将 `perspective` 声明在 `.player-container` 上，是**结构语义最准确、渲染范围最精确、性能最优**的写法。

若声明在 `body` 上会扩大透视作用域，不符合模块化设计原则；

若直接写在 `.player` 上则无法形成有效观察空间，无法产生正确透视。

##### `transform-style: preserve-3d` 

用于**建立三维嵌套上下文**的关键属性。

规范定义：

- 默认值 `flat`：子元素的三维变换会被压扁在父元素的二维平面上，不保留真实空间关系
- `preserve-3d`：允许子元素**在三维空间中独立渲染**，保留层级、旋转、景深等真实三维关系

`.player` 是**多层视觉元素的容器**，并且自身应用了 `rotateY` / `rotateX` 等 3D 变换。

为了使其内部所有子元素能够**跟随父容器共同参与三维空间渲染**，必须在 **`.player` 本身** 上声明：
