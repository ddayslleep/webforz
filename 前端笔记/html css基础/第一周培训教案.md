#### 补充说明：

在 HTML5 技术层面，标准并未严格禁止页面使用多个 `<h1>` 标签，在嵌套的 `<section>` 等元素中分别使用是可行的。但“可行”不等于“推荐”，行业最佳实践始终是“一页一 h1标签”。

1.  **对搜索引擎**：唯一的 H1 标签能清晰传递页面核心主题，避免因多个 H1 导致主题分散，从而削弱关键词关联性与排名。
2.  **对用户**：唯一的 H1 标签作为最醒目的主标题，能帮助用户快速理解页面内容，并构建清晰的章节层级（H2、H3），改善阅读体验。

##### `div` 和 `section` 对比

- **`<section>`**：它是一个**语义化标签**。它的含义是“文档或应用的一个通用区块”，比如文章的一个章节、对话框的选项卡、页面中的一组相关内容。它明确地告诉浏览器和开发者：“这里是一个**独立的内容区块**”。
- **`<div>`**：它是一个**无语义标签**。它的含义就是“一个分隔”或“一个容器”。它只用于**结构布局和样式化**（比如用CSS来定位、设置背景颜色、排版），它本身不传达任何关于其内部内容性质的信息。

在 HTML5 中，每个 `<section>` 定义了一个独立的内容区块。在这个区块内部，你可以使用一个 `<h1>` 作为**这个区块自身的标题**。这个 `<h1>` 的“权重”或“重要性”是**局限在这个 `<section>` 内部**的。

```html
<!-- 标准且允许的 HTML5 写法 -->
<section>
  <h1>区块一的标题</h1>
  <p>区块一的内容...</p>
</section>
<section>
  <h1>区块二的标题</h1>
  <p>区块二的内容...</p>
</section>

<!-- 不推荐/无意义的写法 -->
<div>
  <h1>区块一的标题</h1> <!-- 这里的 h1 就成了页面级的主标题，而不是区块一的标题 -->
  <p>区块一的内容...</p>
</div>
<div>
  <h1>区块二的标题</h1> <!-- 这里又出现一个页面级的主标题，违背最佳实践 -->
  <p>区块二的内容...</p>
</div>
```

当你使用 `<div>` 时，浏览器和搜索引擎**无法知道**这两个 `<div>` 是两个独立的内容区块。它们只会看到：**同一个页面里，出现了两个没有层级关系的 `<h1>` 标签**。这直接就回到了我们之前讨论的“多个 H1”的问题，会削弱页面主题的清晰度。

而使用 `<section>` 时，你是在向机器（浏览器、搜索引擎）明确声明这里开始了一个新的话题

| 特性                 | `<section>`                                  | `<div>`                                                      |
| :------------------- | :------------------------------------------- | :----------------------------------------------------------- |
| **语义**             | **有**。代表一个独立的内容区块。             | **无**。纯粹用于布局和样式。                                 |
| **对SEO影响**        | **有**。帮助搜索引擎理解页面结构和内容分区。 | **无**。搜索引擎会忽略它作为结构信息。                       |
| **使用场景**         | 当内容自然地构成一个章节、一个主题单元时。   | 当需要用一个容器来包裹内容以应用CSS样式或JS功能，且不增加任何语义时。 |
| **能否创建文档大纲** | **能**。它是 HTML5 大纲算法的一部分。        | **不能**。它不参与创建文档大纲。                             |

最佳实践建议

1. **默认使用 `<div>`**：当你只是需要一个容器来给内容设置样式、做布局、或绑定事件时，`<div>` 是最合适的选择。它干净、通用、无副作用。
2. **有理由时使用 `<section>`**：当你包裹的这段内容，在逻辑上构成了文档的一个章节、一个独立的话题，并且你**希望这个事实被机器（搜索引擎、屏幕阅读器）所理解**时，就应该使用 `<section>`。

#### label标签说明

在html中，<label> 标签通常和<input> 标签一起使用，标签为input元素定义标注(标记)。label标签和特定表单控件关联之后，当用户点击标签中的文本时，浏览器就会自动将焦点转到和该标签相关联的控件上。（增大点击范围）

## 1.3 CSS三大特性

### 一、CSS层叠性

层叠性是多种CSS样式的叠加，是浏览器处理样式冲突的方式。在HTML中对于同一个元素可以有多个CSS样式存在，当有相同权重的样式存在时，会根据这些样式出现的先后顺序来决定，处于最后面的CSS样式将会覆盖前面的CSS样式。

```css
<style>
h1{
	color: blue;
}
h1{
    color: red;
    /* 显示的字体颜色为红色 */
}
</style>
```

### 二、CSS继承性

简单的理解为-“子承父业“，是指子标签会继承父标签的文字控制属性，如文本颜色和字号。想要设置一个可继承的属性，只需将它应用于父元素即可。

```html
<div class="father">
	<p>son of div</p>
</div>
<style>
.father{
	color: blueviolet;
}
</style>
```

son of div 会继承father类的颜色样式，显示为blueviolet。

什么时候不继承？

### 三、CSS优先级

在复杂CSS样式表，往往并不是相同样式不同值的堆叠这么简单，经常出现两个或多个不同样式规则应用在同一元素上，这时到底采用哪个样式呢？这就是典型的CSS优先级问题。 处理优先级问题，就是考虑样式权重的高低。这里先给大家介绍一些特殊的情况:

1. 继承样式的权重为0：也就是说，在嵌套结构中，无论父元素样式权重多大，子元素继承时，应用在子元素上的权重都为0，即子元素定义的样式会覆盖所有继承来的样式。

2. 行内样式优先：应用style属性的元素，其行内样式的权重非常高，可以理解为远大于100。

3. 权重相同时，CSS遵循就近原则。也就是说靠近元素的样式具有最大的优先级，或者说排在最后的样式优先级最大。

4. CSS定义了一个!important命令，该命令被赋予最大的优先级。也就是说不管权重如何以及样式位置的远近，!important都具有最大优先级。

|选择器类型|权重值|
|---|---|
|继承或者*|0|
|类选择器、伪类选择器、属性选择器|10|
|ID选择器|100|
|行内样式|1000|
|!important重要性|最高（凌驾于所有权重之上）|

## 1.4 常用选择器的主要类型及说明

### 1.4.1 基础选择器

#### 标签选择器

针对一类标签

```css
h1{
    color: blue;
    font-family: 'Courier New', Courier, monospace;
}
```

#### 类选择器

针对你想要的所有标签使用

```html
<div class="bg"></div>
.bg{
    width: 400px;
    height: 200px;
    background-color: aliceblue; 
}
```

##### 类选择器的其它用法

多类选择器

```html
<div class="class1 class2 class3">这是一个多类选择器示例</div>
<style>
.class1 {
	color: red;
}
.class2 {
	font-size: 20px;
}
.class3 {
	background-color: yellow;
}
</style>
```

#### ID选择器

针对某一个特定的标签使用，一般配合js使用

```html
<p id="text">hahaha</p>
#text{
    width: 400px;
    height: 200px;
    background-color: aliceblue; 
}
```

#### 通用选择器(通配符)

针对所有的标签都适用

```css
* {
    margin: 0;
    padding: 0; 
}
```

### 1.4.2 组合选择器

#### 后代选择器(空格分隔)

选择指定元素内部的==所有符合条件的后代元素==。描述: 通过空格分隔两个选择器，以选择第一个选择器的后代中符合第二个选择器的元素。示例: div p 会选择所有<div> 元素内的<p> 元素。

```css
div p {
	color: blue;
}
```

#### 子元素选择器(> 分隔)

选择指定元素的==直接子元素==。描述: 使用> 符号分隔选择器，通过> 符号选择直接子元素。示例: div > p 会选择所有直接位于<div> 元素内的<p> 元素。

```css
div > p {
	color: red;
}
```

#### 群组选择器/并集选择器(,分隔)

允许同时选择多个元素，并将相同的样式应用于它们。描述: 允许同时选择多个选择器，并对它们应用相同的样式规则。群组选择器通过逗号, 分隔不同的选择器。示例: h1, h2, p { color: blue; } 会将<h1> 、<h2> 和<p> 元素的文字颜色都设置为蓝色。

```
h1, h2, p { 
	color: blue; 
}
```

#### 交集选择器(选择器之间连写，无任何符号)

同时满足两个标签的情况

```css
p.box{
	color:red;
}
```

注：标签选择器必须写在最前面

#### 补充选择器：

##### 相邻兄弟选择器(+ 分隔)

==选择与指定元素在同一层级且紧接在它后面的元素==。描述: 使用+ 符号分隔选择器，选择紧接在另一元素后的元素，且二者有相同的父元素。示例: h2 + p 会选择紧跟在<h2> 元素之后的<p> 元素。

```css
h2 + p {
	color: green;
}
```

**场景举例**：

-   **场景一：为标题后的第一段增加特殊样式**（常见于文章排版）

    ```css
    h2 + p {
      margin-top: 0;
      font-weight: bold;
    }
    ```

    -   只有**紧跟在 `</h2>` 后面的第一个 `<p>`** 会被选中，其它段落不受影响。
    -   如果 `h2` 后面跟的不是 `<p>`，而是 `<div>`，则这个规则无效。

-   **场景二：表单里错误提示的位置控制**

    ```css
    input.error + .error-msg {
      display: block;
    }
    ```

    只有当 `.error-msg` 这个提示框**紧跟在** `input.error` 后面时，它才会显示。

-   **场景三：列表项之间的分割线（去掉第一个的边框）**

    ```css
    li + li {
      border-top: 1px solid #ccc;
    }
    ```

    这个写法非常经典：选中**从第二个开始**的 `li`，给它们加上上边框。

##### 通用兄弟选择器(~ 分隔)

==选择与指定元素在同一层级的所有后续兄弟元素==。描述: 使用~ 符号分隔选择器，选择某个元素之后的所有兄弟元素(共享相同父元素)，而不仅仅是紧接在后面的元素。示例: h2 ~ p 会选择所有与h2 元素在同一层级的p 元素。

```css
h2 ~ p {
	color: black;
}
```

**场景举例**：

-   **场景一：鼠标悬停时改变后面所有元素的状态**

    ```css
    .item:hover ~ .item {
      opacity: 0.5;
    }
    ```

    当鼠标划过某个 `.item` 时，它**后面**所有的 `.item` 都会变透明（类似图片画廊的联动效果）。

-   **场景二：表单中控制后续输入框的可用性（配合 `:checked`）**

    ```css
    input:checked ~ input {
      background-color: #f0f0f0;
    }
    ```

    如果某个 `radio` 或 `checkbox` 被选中，它**后面**的所有输入框都变灰。

-   **场景三：根据状态切换不同区域的样式**

    ```css
    #notice:checked ~ .notice-box {
      display: block;
    }
    #notice:checked ~ .other-content {
      color: green;
    }
    ```

    一个勾选框可以控制多个后面兄弟元素的样式。

```html
<style>
    /* 相邻兄弟选择器：紧跟在 h2 后面的 p */
    h2 + p {
        color: green;
        font-weight: bold;
    }

    /* 通用兄弟选择器：h2 后面所有的 p */
    h2 ~ p {
        background-color: #f0f0f0;
    }

    /* 为了让效果更明显，添加一些基础样式 */
    body {
        font-family: Arial, sans-serif;
        padding: 20px;
    }

    .note {
        color: #666;
        font-size: 14px;
        margin-top: 20px;
        padding: 10px;
        background-color: #fff3cd;
        border-left: 4px solid #ffc107;
    }
</style>

<h1>兄弟选择器演示</h1>
<h2>标题1 - 我是 h2</h2>
<p>这是紧跟在 h2 后面的 p（同时被 + 和 ~ 选中）</p>
<p>这是 h2 后面的第二个 p（只被 ~ 选中）</p>
<div>这是一个 div（不是 p，不受影响）</div>
<h2>标题2 - 我是另一个 h2</h2>
<p>这是紧跟在第二个 h2 后面的 p（同时被 + 和 ~ 选中）</p>
<p>这是第二个 h2 后面的第二个 p（只被 ~ 选中）</p>
```

### 1.4.3 伪类选择器

#### 动态伪类选择器

- `:hover`：鼠标悬浮到元素上时触发（悬浮态）。
- `:focus`：选择获得焦点的元素，如通过键盘或鼠标点击选中的元素。当元素获得焦点时触发，常见于输入框、按钮等。
- `:visited`：选择已访问过的链接（访问后）。
- `:link`：选择未访问过的链接（访问前）。
- `:active`：点击时（激活）

#### 结构伪类选择器

基于元素在文档树中的位置或与其他元素的关系来选择元素，选择文档树中特定结构的元素。

- `:first-child`：选择父元素的第一个子元素。
- `:last-child`：选择父元素的最后一个子元素。
- `:nth-child(n)`：选择父元素的第n个子元素。（常用2n/2n+1）
- `:nth-last-child(n)`：类似于`:nth-child(n)`，但它是从最后一个子元素开始计数的。
- `:first-of-type`：选择父元素下相同类型元素的第一个子元素。
- `:last-of-type`：选择父元素下相同类型元素的最后一个子元素。
- `:nth-of-type(n)`：选择父元素下相同类型元素的第n个子元素。
- `:nth-last-of-type(n)`：类似于`:nth-of-type(n)`，但它是从相同类型元素的最后一个子元素开始计数的。

``:nth-child(n)`中的n如果算出来是负数会怎样？

答：`:nth-child(n)` 中的 `n` 若为负数，公式结果 ≤ 0 的元素不会被选中。
例如 `:nth-child(-n+0)` 无匹配，`:nth-child(-n+2)` 匹配第 1、2 个

#### 补充伪类选择器：

##### 表单伪类选择器

用于表单元素，根据表单元素的状态来选择元素。

- `:checked`：选择被选中的表单元素，如单选框、复选框等。

- `:enabled`：选择启用的表单元素。

- `:disabled`：选择被禁用的表单元素。

- `:read-only`：选择只读的表单元素。

- `:read-write`：选择可写的表单元素。

##### 目标伪类选择器

- `:target`：选择当前活动的目标元素，即URL的片段标识符（#后面的部分）指向的元素。

##### 否定伪类选择器

- `:not(selector)`：选择不符合特定选择器的元素。允许你选择不匹配该选择器的所有元素，它可以用在几乎任何选择器中，来排除某些不需要的元素。

    </section>
    
    <!-- 目标伪类选择器演示 -->

        <a href="#target1">点击跳转到目标1</a> | 
        <a href="#target2">点击跳转到目标2</a>

### 1.4.4 伪元素选择器

伪元素选择器（::开头）: 用于选择元素的某个部分或向元素添加虚拟内容，并允许为这些虚拟内容或特定部分设置样式。

#### 文本相关伪元素

- `::first-letter`：用于选择文本块的第一个字母，并允许为其设置特殊的样式。这常用于实现首字下沉或特殊首字母效果。

- `::first-line`：用于选择文本块的第一行，并允许为其设置与后续文本不同的样式。这常用于强调段落或文本块的开头部分。

#### 内容生成伪元素

- `::before`：在元素的内容之前插入新的内容或装饰。这通常与content属性一起使用，以定义要插入的具体内容。

- `::after`：在元素的内容之后插入新的内容或装饰。同样，这也需要content属性来指定要插入的内容。

示例： `p::before { content: "注意:"; color: red; }` 会在每个 <p> 元素的内容前添加“注意:”字样，并设置其颜色为红色。

## 1.6 背景

### 1.6.1 背景颜色

语法：`background-color : 合法颜色值;` （用于设置元素背景颜色）

### 1.6.2 背景图片引入

语法：`background-image : url(img/02.png);` （用于设置元素背景图片，url中填写图片路径）

### 1.6.3 背景图片的定位与尺寸

#### 背景图片定位（background-position）

取值：

1. x y 分别设置x轴和y轴：以px为单位的数字；关键字（x：left/center/right，y：top/center/bottom）

2. 一个值：只设置x轴，y轴默认居中

3. 英文关键字可以先写垂直方向再写

#### 背景图片尺寸（background-size）

取值：

1. 两个值：设置宽高，以px为单位的数字

2. 一个值：同时设置宽高

3. contain：图片根据元素尺寸自动等比缩放，正好全部显示图片

4. cover：图片根据元素尺寸自动等比缩放，正好把元素铺满

### 1.6.4 背景图片平铺（background-repeat）

- `repeat`：默认值，整体平铺
- `no-repeat`：不平铺
- `repeat-x`：沿x轴平铺
- `repeat-y`：沿y轴平铺

### 1.6.5 背景图片固定（background-attachment）

属性值：`fixed`

fixed定位，所谓的“窗口”，是什么窗口？是哪一部分区域？浏览器的区域有哪些？

答：

- `fixed` 定位的“窗口”指**浏览器可视区域（viewport）**，不包括滚动条、工具栏。
- 浏览器区域包括：可视区（滚动区域）、地址栏、标签栏、书签栏、滚动条。`fixed` 元素相对于**可视区**固定，滚动时位置不变。

### 1.6.6 背景简写方式

- 常规简写：`background：color img repeat position；`（注意：简写方式中不包含size，size需单独书写）

  顺序固定吗？

- 最简方式：`background：color / img；`
